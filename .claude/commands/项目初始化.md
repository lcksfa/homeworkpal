为"作业搭子"RAG智能问答系统执行完整的初始化流程。仅在项目首次创建时使用，用于搭建基于人教版教材的三年级学生AI作业助手环境、生成工程护栏文件和建立开发规范。

执行内容：
1. 创建Python虚拟环境(v3.11+)并使用uv安装依赖
2. 初始化PostgreSQL向量数据库(pgvector)和Docker环境
3. 生成init.sh启动脚本（支持--no-frontend, --reset-vectordb, --env, --load-docs等参数）
4. 创建feature_list.json中文化功能清单（按5个开发阶段组织，40项功能）
5. 建立claude-progress.txt结构化进度追踪文件
6. 创建标准Python包结构src/homeworkpal/和pyproject.toml配置
7. 初始化git仓库并完成记录详实的首次提交
8. 生成README文档包含会话启动仪式和开发流程说明

关键工件：
- init.sh：一键启动开发环境脚本（Docker PostgreSQL + 服务管理）
- feature_list.json：中文化端到端功能测试清单（Task-X.Y + RAG系列）
- claude-progress.txt：多会话开发进展记录文件
- pyproject.toml：完整依赖管理和构建配置
- src/homeworkpal/：标准Python包结构
- README.md：项目说明和技术栈文档

技术栈配置：
- 后端：FastAPI + Uvicorn + SQLAlchemy
- 数据库：PostgreSQL + pgvector + timescale-vector
- 前端：Chainlit交互式Web界面
- AI引擎：阿里云通义千问(Qwen) + DeepSeek
- 嵌入模型：BAAI/bge-m3（中文优化）
- 包管理：uv现代化Python包管理器

输出要求：
- 确保环境可重现，init.sh具有幂等性和Docker容器管理
- 所有初始功能标记为未通过状态（passes=false）
- 功能清单按开发阶段组织，严格按Phase 1-5顺序执行
- 提供清晰的下一步开发指导和环境要求说明
