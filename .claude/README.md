## 核心设计理念

### 1. 拆解：化繁为简
将复杂的作业搭子系统拆解为35项明确的功能清单，每个功能包含：
- 清晰的三年级教育场景描述
- 详细的验证步骤（涵盖文档处理、检索、生成、批改等）
- 明确的通过标准（基于教学效果和用户体验）
- 严格的治理规则（禁止删除功能，只能标记废弃）

### 2. 留痕：全程可追溯
每次会话都必须留下结构化记录：
- `claude-progress.txt`：作业搭子会话进展日志
- Git提交记录：原子化的变更历史
- 测试证据：E2E验证的截图和日志（包含学生使用场景）
- 性能指标：作业批改响应时间和准确性的量化数据

### 3. 自证：用户视角验证
强制要求以学生/家长视角进行端到端测试：
- 浏览器自动化测试（模拟学生上传作业图片）
- 完整作业批改旅程模拟（拍照→识别→批改→记录错题）
- 真实教育场景下的功能验证（基于人教版教材）
- 客观的教学质量评估标准（鼓励性语言、知识点溯源）

### 4. 可回滚：安全保护机制
确保所有变更可快速回滚：
- 最小化变更范围（每次只实现一个功能）
- 原子化提交策略（确保每个提交都是可回滚的）
- 快速恢复流程（保护学生数据和学习记录）
- 事故记录和预防机制（确保教育连续性）

## Homework Pal功能开发流程

### 人教版教材文档处理
1. **教材上传**：支持人教版三年级教材PDF、TXT、MD格式
2. **内容解析**：按单元-课题-知识点层级解析教材内容
3. **智能分段**：基于教学结构的精细化文档分割
4. **向量化处理**：使用BGE-M3模型生成高质量向量表示
5. **索引建立**：构建支持教材溯源的检索索引

### 作业批改检索功能
1. **题目理解**：分析学生作业题目和考查知识点
2. **向量搜索**：基于语义相似度检索相关教材内容
3. **结果排序**：结合知识点相关性和年级适配度排序
4. **上下文过滤**：根据三年级学习进度调整检索范围
5. **结果优化**：确保检索内容适合三年级理解水平

### 鼓励性生成功能
1. **上下文构建**：组织教材知识点和鼓励性语言模板
2. **启发式引导**：生成符合教学理念的批改建议
3. **知识点溯源**：准确标注人教版教材出处和页码
4. **质量控制**：确保语言适合三年级学生心理特点
5. **交互优化**：保护孩子学习兴趣的友好界面

## 开发规范

### 会话启动仪式
每个开发会话必须遵循标准流程：
1. `pwd` → 确认工作目录
2. `git log` → 查看变更历史
3. `claude-progress.txt` → 阅读进展记录
4. `./init.sh` → 启动开发环境
5. `E2E冒烟测试` → 验证系统健康
6. 选择目标功能 → 开始开发

### 功能实现原则
- **单次一事**：每次会话只实现一个功能
- **最小变更**：保持改动范围可控
- **用户验证**：必须通过E2E测试
- **干净收尾**：完整记录和提交

### 质量标准
- 所有功能必须通过浏览器自动化测试
- 代码变更必须有对应的测试覆盖
- 提交信息必须包含详细的变更说明
- 进展记录必须包含充分的上下文信息

## 故障排除

### 常见问题解决
1. **环境启动失败**：检查Python版本和依赖配置
2. **向量数据库连接问题**：验证数据库服务状态和配置
3. **文档处理异常**：检查文件格式和权限设置
4. **检索结果质量差**：调优向量化参数和分段策略
5. **生成回答不准确**：优化Prompt模板和上下文管理

### 紧急恢复流程
1. 使用`/rag回滚恢复`快速恢复到稳定状态
2. 分析问题根本原因并记录
3. 修复问题后重新验证系统
4. 更新预防措施和检查清单
