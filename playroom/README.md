# Playroom 测试区域

## 🚪 重要规则
这个目录专门用于临时测试和实验文件。

### 使用指南
- ✅ 可以在这里创建任何测试文件
- ✅ 可以进行实验性代码编写
- ✅ 可以测试新功能而不影响主项目
- ❌ 完成测试后请及时清理文件
- ❌ 不要把生产代码放在这里

### 文件命名建议
- 使用前缀：`test_`, `experiment_`, `demo_`
- 包含日期：`test_pdf_processing_2025-12-10.py`
- 描述性名称：`experiment_vector_search.py`

### 清理规则
- 测试完成后立即删除临时文件
- 如果测试结果有价值，可以移动到正式目录
- 定期清理过期的测试文件

### 示例文件
```
playroom/
├── test_pdf_processing.py     # PDF处理测试
├── experiment_embedding.py    # 向量嵌入实验
├── demo_api_call.py          # API调用演示
└── README.md                 # 本文件
```

记住：保持项目根目录的整洁，所有临时测试都在这里进行！
