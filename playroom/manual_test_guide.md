# 手动测试指南：作业搭子问答功能集成

## 🎯 测试目标
验证前端Chainlit界面已成功集成RAG问答功能，用户体验流畅。

## 🔧 环境准备
- ✅ 前端服务：http://localhost:8000 (Chainlit)
- ✅ 后端API：http://localhost:8001 (FastAPI)
- ✅ 问答功能：已集成到前端界面

## 📋 测试步骤

### 1. 访问前端界面
```
打开浏览器，访问：http://localhost:8000
```
**期望结果**：
- 页面正常加载，显示小栗子老师的欢迎消息
- 显示4个功能按钮：🤔 提问学习、📸 检查作业、📅 整理清单、📕 复习错题
- 提示可以直接在聊天框输入问题

### 2. 测试基本问候功能
```
在聊天框输入：你好
```
**期望结果**：
- 立即返回简单问候回复，不显示"正在思考"
- 回复内容："你好呀！我是小栗子，有什么学习问题想问我吗？🌰"
- 包含示例问题提示

### 3. 测试帮助功能
```
在聊天框输入：帮助
```
**期望结果**：
- 立即返回帮助信息，不显示"正在思考"
- 显示功能列表和示例问题
- 不调用后端API

### 4. 测试问答功能 - 数学问题
```
在聊天框输入：什么是加法？
```
**期望结果**：
- 首先显示"🤔 小栗子正在认真思考你的问题...让我从教材里找找答案！⏳"
- 等待5-10秒后，显示格式化答案：
  - 💡 **答案**：基于教材的详细解释
  - 📚 **教材来源**：（如果有教材内容）
  - ✨ **小栗子说**：鼓励性和学习指导

### 5. 测试问答功能 - 语文问题
```
在聊天框输入：怎么写好作文？
```
**期望结果**：
- 显示"正在思考"提示
- 返回基于人教版语文教材的作文指导
- 包含教学建议和鼓励话语

### 6. 测试学科自动检测
```
测试输入：
- "数学中的减法是什么？" → 应自动检测为数学
- "语文学习要注意什么？" → 应自动检测为语文
- "英语单词怎么记？" → 应自动检测为英语
```

### 7. 测试错误处理
```
测试场景：
1. 后端API不可用时的错误提示
2. 网络超时时的友好提示
3. 非问题输入的引导提示
```

### 8. 测试界面交互
```
点击"🤔 提问学习"按钮
```
**期望结果**：
- 显示使用指导消息
- 提供示例问题
- 引导用户直接在聊天框输入

## 🔍 关键验证点

### ✅ 功能正确性
- [ ] 问候和帮助功能在前端处理，不调用API
- [ ] 问筢单正确调用后端API
- [ ] 学科自动检测准确
- [ ] 答案基于人教版教材内容

### ✅ 用户体验
- [ ] "正在思考"提示及时显示和移除
- [ ] 答案格式清晰，包含emoji图标
- [ ] 语言风格适合三年级学生
- [ ] 包含鼓励性和教育性指导

### ✅ 错误处理
- [ ] API调用失败时友好提示
- [ ] 网络问题时有合理建议
- [ ] 非问题输入时正确引导

### ✅ 响应性能
- [ ] 问候/帮助立即响应（<1秒）
- [ ] 问答响应时间合理（5-15秒）
- [ ] 界面交互流畅无卡顿

## 🎯 验收标准
1. **核心功能**：用户可以直接在前端聊天框输入学习问题，获得基于人教版教材的答案
2. **界面友好**：保持小栗子老师的亲切形象，答案格式清晰，包含教材来源和鼓励话语
3. **错误处理**：API调用失败时有合适的错误提示，网络问题时给出合理建议
4. **响应性能**：问答功能响应时间在合理范围内，用户体验流畅
